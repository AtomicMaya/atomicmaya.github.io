<h4 id='baby-shark'>Baby Shark (50 points)</h4>

<p style='color: lime;'>During my holiday in Bahamas, I met a baby shark. The shark wanted to sing me something but couldn't. Can you sing that for me?</p>

<p style='color: lime;'>Flag Format: {`KCTF{SomE_T3xT_h3r3}`}</p>

<br/>

Once the JAR is downloaded, go to <ExternalLink url='httpd://javadecompilers.com' text='javadecompilers.com' />, and upload it there.

We obtain a ZIP which we then unpack, revealing the following file structure:

<ImageContainer image={img_tree} style="width: auto; height: 150px;" alt="File tree" />

We can see a few interesting Java files, which we'll get into quickly.

The first is `Flag.java`, it is not very interesting:

<ImageContainer image={img_flagcode} style="width: 80%; height: auto;" alt="A few false flags." />

In `Strings.java` we can see what resembles strings that are Base64 encoded:

<ImageContainer image={img_stringscode} style="width: 80%; height: auto;" alt="A few false flags." />

If we throw these in a Base64 decoder, the last gives us the flag: <code>{`KCTF{7H15_W@5_345Y_R16H7?}`}</code>

<h4 id='droid-flag'>Droid Flag (100 points)</h4>

One needs to decompile the provided APK, and understand what the decompiled code does, to retrieve the flag.

Once the ZIP is downloaded, extract the APK contained within, then go to <ExternalLink url='httpd://javadecompilers.com' text='javadecompilers.com' />, and upload it there.

We obtain a ZIP which we then unpack, revealing the following file structure:

<ImageContainer image={img_droidtree} style="width: auto; height: 250px;" alt="Decompiled file tree." />

We can see a few interesting Java files, which we'll get into quickly.

The first is `MainActivity.java`, it contains a few interesting lines:

<ImageContainer image={img_mainactivity} style="width: 80%; height: auto;" alt="The code for MainActivity.java." />

We can see that this code calls a number of functions in the `StringHandler` class from the `StringHandler.java` file, in the following order:

<code>{`$(S1){$(S3)_$(S2)_$(S4)}`}</code>

<ImageContainer image={img_stringhandler} style="width: 80%; height: auto;" alt="The code for StringHandler.java." />

- `S1` gets a value defined in class `C074SR` where the identifier is `f112s5`
- `S2` gets a value defined in class `C074SR` where the identifier is `f113s6`
- `S3` gets a value defined in class `C074SR` where the identifier is `f114s7`
- `S4` gets a value defined in class `C074SR` where the identifier is `f115s8`

Those values are:

<ImageContainer image={img_c0754code} style="width: 80%; height: auto;" alt="The code for class C0754RCode." />

We look for any identifier with those numbers (`grep` is useful), and find some in `public.xml`:

<ImageContainer image={img_publicxml} style="width: 80%; height: auto;" alt="The public.xml file." />

We then look for the names provided in `strings.xml`:

<ImageContainer image={img_stringsxml} style="width: 80%; height: auto;" alt="The strings.xml file." />

In `StringHandler.java` we see that the values for `S2`, `S3` and `S4` are then reversed, which makes the final flag:<br/><code>{`$(S1){$(S3)_$(S2)_$(S4)}`}</code> -> <code>{`KCTF{aNdR01D_s1MpL3_r3V3rS3}`}</code>
